# Kitchenham Guidelines Core Notes

Derived from the user-provided local PDFs `Kitchenham 2019.pdf` and `Kitchenham-2007Systematicreviews5-8.pdf`, which both contain the EBSE-2007-01 guideline content.

## What this source governs

This is the main methodology source for planning, conducting, and reporting systematic literature reviews in software engineering.

## Major sections visible in the local source

- reasons for performing systematic literature reviews
- importance, advantages, and disadvantages of SLRs
- other review types including systematic mapping studies and tertiary reviews
- the review process
- planning
- research questions
- review protocol development

## Methodological expectations to derive from this source

- define the need for the review before designing the protocol
- formulate explicit research questions
- distinguish review type according to objective
- document protocol decisions before conducting the review
- keep the process reproducible and auditable
- separate planning, conducting, and reporting concerns

## Review-type guidance

- use `SLR` for focused questions and evidence synthesis
- use `SMS` for broader categorisation and landscape mapping
- use `tertiary review` when synthesising existing secondary studies

## Agent behavior to enforce

- if the user asks for a protocol, include review objective, questions, eligibility, sources, search logic, selection process, quality handling, extraction fields, synthesis plan, and validity considerations
- if the user asks for a review write-up, keep the method traceable rather than rhetorical
- do not claim a systematic review unless the workflow described is genuinely systematic
