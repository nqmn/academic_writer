# Kitchenham and Brereton 2013 Process-Research Notes

Derived from the user-provided local PDF `Kitchenham, Brereton - 2013 - A systematic review of systematic review process research in software engineering.pdf`.

## Why this source matters

This source reviews software-engineering research about the SR process itself. It is useful when the agent needs to justify process refinements rather than simply repeat the 2007 guideline.

## Recurring problems highlighted in the local source

- systematic reviews take a long time
- software-engineering digital libraries are not well suited to broad searches
- quality assessment across heterogeneous empirical methods is difficult

## Recommendations visible in the local source

- avoid over-relying on structured questions to construct search strings
- use a quasi-gold standard based on a limited manual search to help construct and evaluate search strings
- textual analysis tools may help with search-string construction and inclusion or exclusion decisions, but need careful evaluation
- tools to manage the SR process are useful, but should not be trusted uncritically

## Agent behavior to derive from this source

- when proposing search strings, prefer iterative refinement over rigid one-shot construction
- when discussing search quality, mention the value of a quasi-gold standard or pilot search set
- when evaluating methods, acknowledge that quality assessment remains difficult across mixed empirical designs
- avoid promising automation as a substitute for methodological judgment
