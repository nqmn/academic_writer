# The Matrix Method of Literature Review

**Citation:** Klopper, R., Lubbe, S., & Rugbeer, H. (2007). The matrix method of literature review. *Alternation, 14*(1), 262–276. ISSN 1023-1757.

**Keywords:** Concept matrix, epistemology, knowledge, literature review, matrix analysis, problem-based research

---

## Core Argument

The concept matrix method provides a systematic, objective tool for literature review at the beginning of problem-based research. It prevents subjective "patchwork quilt" reviews or cherry-picking of supporting references while ignoring contrary ones.

---

## Epistemological Foundation: Four Levels of Competency

The matrix method helps researchers progress through a knowledge competency continuum:

```
Unconscious incompetence
        ↓
Conscious incompetence
        ↓
Conscious competence
        ↓
Unconscious competence
```

| Stage | Description |
|-------|-------------|
| **Unconscious incompetence** | Unaware of the topic and its relationships |
| **Conscious incompetence** | Aware of not knowing; no systematic plan |
| **Conscious competence** | Beginning to hypothesize; not yet fluent |
| **Unconscious competence** | Expert-level knowledge; can identify interrelationships intuitively |

The matrix method rapidly moves researchers from **conscious incompetence** to **conscious competence**.

---

## What Is a Concept Matrix?

A matrix is a **rectangular array of symbols arranged in rows and columns** — a conceptual framework in table/grid format that:
- Enables conclusions about non-obvious relationships
- Contains verbal information, quotes, summaries, extracted data
- Provides an "eagle's eye" overview of literature

**Layout:**
- **Rows** = references (listed in abbreviated Harvard style)
- **Columns** = key concepts derived from the problem statement
- **Cells** = tick (✓) if the concept is discussed in that reference

---

## How to Build a Concept Matrix

1. **Formulate a clear problem statement** — extract key concepts from it
2. **Use those concepts as column headers** in the matrix
3. **Survey abstracts and summaries** (not full texts) to identify relevant references
4. **Tick cells** where a concept appears in a reference
5. **Conduct critical comparative analysis** of all references under each concept column
6. **Write the literature review** using the concept columns as section headings

> The problem statement itself becomes the filter that ensures only relevant literature is included.

---

## Benefits of the Concept Matrix Approach

- Liberates the reviewer from being trapped within individual references
- Prevents mere paraphrase; enables comparative critical analysis
- Allows supervisors to verify relevance at a glance
- Establishes whether sufficient references exist under each concept
- Prevents unintentional plagiarism by tracking sources systematically
- References NOT on the matrix should not appear in the bibliography

---

## Matrix Reorganization Options

The matrix can be ordered to foreground different knowledge aspects:

| Matrix Type | Purpose |
|-------------|---------|
| Time-ordered matrix | Longitudinal/historical aspects (publication date) |
| Role-ordered matrix | Perspectives of role occupants on a specific issue |
| Checklist matrix | Integrated data on a summative index or scale |
| Conceptually clustered matrix | Organized around a central theme |
| Effects matrix | Outcomes and dependent variables |
| Site dynamics matrix | Present forces, processes, and outcomes in context |
| Event listing | Series of events in any possible order |
| Causal network | Interrelationships between dependent/independent variables |

## Three Problematic Cases (What to Avoid)

**Case 1** — Student could not stop; produced 180-page literature review with no defined scope. Represents **conscious incompetence** — no problem statement or key concept filter.

**Case 2** — Student claimed grounded theory to avoid literature review, then produced 100 pages of theory. Represents **unconscious incompetence** — did not understand when grounded theory is appropriate (only when the field is truly emergent with insufficient literature).

**Case 3** — Student could not organize literature review under structured headings; produced an unsystematic document. Lacked conceptual scaffolding.

---

## Aims of a Literature Review (per the Matrix Method)

1. Establish which research problems have already been solved (remove them from the research equation)
2. Give readers a clear idea of the research's nature and context
3. Convince readers of the researcher's knowledge of the field
4. Build a case for the empirical part of the study

---

## Key Precedents for Matrix Analysis

- **Physics**: Heisenberg's Matrix Mechanics; Heisenberg Uncertainty Principle (1927)
- **Linguistics**: Chomsky & Halle (1968); Quirk et al. (1974); Lass (1984)
- **Health Sciences**: Garrard (1999/2004) — popularized matrix review in health sciences
- **Informatics**: Webster & Watson (2002) — concept matrix for IS literature review (concept-centric vs. author-centric)

---

## Key Takeaways for Dissertation/Research Use

- Matrix is **iterative** — update as understanding of the topic grows
- There are **no correct matrices**, only functional ones (Miles & Huberman, 1994)
- The matrix should be included as an **appendix** in the dissertation
- Concept columns in the matrix become **headings** in the literature review
- The review must progress from individual reference critique to **critical comparative analysis** across all references per concept
